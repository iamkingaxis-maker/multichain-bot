## Attack angles that did NOT pan out (verified clean)
- **Fail-open vs fail-closed**: every rug-gate error path lands on `_rg_hf={}` → NEUTRAL → buy allowed. A rugcheck outage can never block all buys. Matches trader.buy's declared posture. (Corollary documented below: outage = fleet effectively UNGATED for 30 min/token.)
- **wait_for timeout vs the cache — no semaphore exists**: the review premise of a "shared single-flight semaphore in `_holder_features_cached`" is wrong — there is none (dip_scanner.py:5618; `_fetch_rugcheck_full` in security/honeypot.py:322 has its own 5s aiohttp timeout, no lock). Nothing to leak. Cache poisoning by cancellation impossible: the cache write happens only after a completed fetch; CancelledError is BaseException on py3.8+ and passes through both `except Exception` clauses. Cost note: no single-flight means concurrent first-buys on one token can duplicate the fetch (egress only, last-write-wins, harmless).
- **Double-fetch / TTL race**: young guard → gate → post-buy stamp are seconds apart on a 30-min TTL — all cache hits after the first; no race. The stamped `rug_gate_buy` verdict is recomputed on the same cached features enforcement saw; if the gate timed out (NEUTRAL) but the stamp fetch succeeds and reads BLOCK, the executed buy is stamped BLOCK — informative, correct.
- **hidden_supply math**: negative hidden (pct sums > 100, real in rugcheck data) can never false-block (rule is `>=`). Double-membership (pool-joined AND insider-tagged holder) subtracts twice → hidden UNDERSTATED → fail-open direction. Partial topHolders (~top-20 only) does overstate "hidden" for fat-mid-tail tokens — but that is the gate's DEFINITION (graded 3.7-4.4% winner-kill), and the grade doc (scratchpad/_hoodlana_class_gate_grade.md) recomputed features with the vault-join fix (fresh fetches), so the shipped code matches the graded pipeline. Early at-entry stamps (n=22) already show the `holders<1000` conjunct missing 2 same-day catastrophics (holders 1553/2057) — exactly what cohort v2's grid re-grade is for; threshold work, not a code bug.
- **Vault-join pool identification**: `markets[].pubkey` + liquidityA/B (+ `*Account` variants) + Raydium V4 authority + tag fallback; guarded for non-str/missing; PumpSwap case (owner == market pubkey) chain-verified on HOODLANA. Clean.
- **rug_gate_verdict two-signal rewrite**: burned LP no longer short-circuits (test-pinned); NEUTRAL only when BOTH signals' inputs absent; all legacy call sites (scan-time shadow line ~23824, live_forward_test, stamp) tolerate the new verdict semantics. Legacy tests pass unchanged.
- **Derisk cap accounting** (278ae83): `sell_fraction` is fraction-of-ORIGINAL everywhere (`sell_slice` clamps to remaining); `derisk_slice` = remaining − cap → post-sale remaining == cap exactly → subsequent ticks no-op; TP1-first makes it a no-op; the pm is informed via `close_position` so ladder accounting stays consistent; full-close guard (`if pool not in st.pos_meta: continue`) present.
- **Depth-gated re-entry across restart**: `exit_book` is in-memory and documented fail-OPEN (both windows are 20 min, same posture as `last_exit`). Deliberate, sound.
- **Feed default-mode identity** (283a2f1): with `RH_FEED_MAX_AGE_H<=24` every aged branch is inert; the only default-path change is stamping `cand[pool]["liq"]` which nothing reads in default mode; defaults are test-guarded (`TestAgedModeDefaultsInert`). Scalp pin (ROSTER[:9]=24h, launch_scalp 20min, aged uncapped) test-pinned. Fresh-pool queue-jump is gated on non-empty queue (backfill flood safe) and default-mode-inert (test-pinned).
- **Quote batch fallback** (278ae83): any transport/shape problem (non-list response, missing tier id, non-200) → None → sequential sweep; zero/revert tiers skipped identically to `_quote_single`. Decimals memoized by lowercased addr; failures NOT cached (18 fallback stays transient). Worst case adds the 10s batch timeout before fallback — latency, never correctness.
- **Uploader orphan / exec semantics** (aa2ffc9): `os.execv` keeps the PID, so the Popen'd uploader remains a child of the lane; lane death = container death = uploader death; Railway restart recreates both. No orphan accumulation. Windows `execv` (parent-returns) is a dev-only quirk; Railway is Linux. Env leakage: execv preserves the full service environment — if shared Railway variables include signing keys the paper-lane process holds them in memory (surface only; the lane never reads keys). Keep the rh-paper-lane service's variable set minimal.
